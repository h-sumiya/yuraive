description: 端末内、SMB、WebDAV、Windows接続からライブラリへ作品を追加します。
---

プレイヤーは、追加したフォルダの中から`.yuraive`と`.yuraive.json`を探してライブラリに表示します。

## 端末内のフォルダ

1. ライブラリの「フォルダを追加」を押します。
2. Android版では「端末」、Windows版では「この PC」を選びます。
3. 作品を含むフォルダを選びます。

作品ごとにフォルダを分け、その上位フォルダをまとめて追加することもできます。

追加したフォルダをライブラリから削除しても、端末内の元ファイルは削除されません。

## SMB

1. 「フォルダを追加」から「SMB」を選びます。
2. 「サーバー」にホスト名またはIPアドレスを入力します。
3. ポートを確認し、「共有名」を入力します。
4. 必要な場合だけドメイン、ユーザー名、パスワードを入力します。
5. 「接続」を押し、表示された一覧から追加するフォルダへ移動します。
6. 「このフォルダを追加」を押します。

既定のSMBポートは`445`です。

ユーザー名とパスワードを両方とも空にすると、guest接続を使用します。

「ライブラリ表示名」は任意で、ライブラリに表示する名前を分かりやすく変えたいときに使います。

## WebDAV

1. 「フォルダを追加」から「WebDAV」を選びます。
2. `https://`で始まるWebDAV URLを入力します。
3. Basic認証が必要な場合だけユーザー名とパスワードを入力します。
4. 「接続」を押し、表示された一覧から追加するフォルダへ移動します。
5. 「このフォルダを追加」を押します。

WebDAVサーバーは、フォルダ一覧に使う`PROPFIND`と、途中からの再生に使うRange付き`GET`に対応している必要があります。

HTTPのURLは使用できないため、HTTPSのURLを指定します。

URL内のユーザー情報、クエリ、フラグメントは使用できません。

認証情報はAndroid版ではAndroid Keystore、Windows版ではWindowsのデータ保護機能で暗号化して保存されます。

端末から到達できれば、Tailscale内のSMBホストやHTTPS WebDAV URLも利用できます。

## Windowsのライブラリ

Android版は、Windows版Yuraiveが共有するライブラリを直接追加できます。

設定とQRコードの手順は[Android版を使う](/getting-started/android/#windowsのライブラリを追加する)で確認してください。

## 追加後に作品が見つからない場合

ライブラリの更新ボタンを押してフォルダを読み直します。

それでも表示されない場合は、次の点を確認します。

- ファイル名が`.yuraive`または`.yuraive.json`で終わっている。
- 選んだフォルダの中に作品ファイルがある。
- JSONの`version`が`1`である。
- 同名の`.yuraive`がある場合は、`.yuraive.json`ではなく`.yuraive`が表示されている。

詳しい対処は[読み込みと再生の問題](/troubleshooting/content/)にあります。
