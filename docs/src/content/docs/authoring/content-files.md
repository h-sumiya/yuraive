---
title: コンテンツのファイルを管理する
description: Yuraiveファイル、素材、スクリプト、レイアウトを安全な相対パスで配置します。
---

Yuraiveでは、`.yuraive.json`があるフォルダをその作品の**コンテンツルート**として扱います。

素材、スクリプト、レイアウトのパスは、すべてYuraiveファイルからの相対パスです。

## フォルダ構成の例

```text
rain-story/
├── graph.yuraive.json
├── audio/
│   ├── opening.ogg
│   └── ending.ogg
├── images/
│   └── cover.webp
├── subtitles/
│   └── opening.vtt
├── scripts/
│   ├── route.star
│   └── button.star
└── ui/
    └── default.yuraive-layout.html
```

ファイルを分けるときは、作品フォルダ内で役割ごとのサブフォルダを作ります。

## パスの規則

JSON、Starlark、レイアウトでは、次の形式でパスを書きます。

```json
{
  "audio": "audio/opening.ogg",
  "subtitle": "subtitles/opening.vtt"
}
```

パス区切りには、Windows上でも`/`を使います。

絶対パス、外部URL、コンテンツルートの外へ出る`..`は使用できません。

ファイル名の大文字と小文字は区別される環境があるため、JSONと実際の名前を同じ表記にします。

## 作品を移動する

作品フォルダ全体を移動すれば、内部の相対パスは変わりません。

素材だけを別のサブフォルダへ動かした場合は、その素材を参照するJSON、スクリプト、レイアウトも更新します。

エディタのファイルツリーで`.star`または`.yuraive-layout.html`を移動すると、開いているグラフ内の参照も更新されます。

OSのファイル管理画面で移動した場合は参照が自動更新されないため、移動後に問題一覧を確認します。

## 作品の識別子

新規作品では、エディタが`metadata.contentId`を自動発行します。

同じContent IDを持つファイルは同じ作品グループとして履歴と統計が集計されます。

ファイル名の変更、フォルダの移動、通常の複製ではContent IDを変える必要はありません。

別作品として履歴と統計を分けたいときだけ、グラフ情報の「新規ID」で再発行します。

## 編集用と配布用の違い

`.yuraive.json`は、スクリプトとレイアウトを別ファイルとして参照する編集用の形式です。

`.yuraive`は、グラフ、Starlark、レイアウトを一つにまとめた配布用の形式です。

配布用ファイルにも音声、動画、画像、字幕は入らないため、相対パスを保ったまま一緒に置きます。

同名の両形式を同じフォルダに置くとプレイヤーは`.yuraive`を優先するため、編集内容の確認時は古い配布用ファイルを取り除くか書き出し直します。
