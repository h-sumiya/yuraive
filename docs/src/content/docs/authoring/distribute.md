---
title: プレイヤー用に配布する
description: グラフ、Starlark、レイアウトを一つの.yuraiveへまとめ、素材と一緒に配布します。
---

完成した作品は、プレイヤー用の`.yuraive`へ書き出せます。

この形式は編集用のグラフ、Starlark、レイアウトを一つのファイルへまとめます。

## 書き出す前に確認する

1. 変更したグラフ、スクリプト、レイアウトを保存します。
2. 問題一覧のエラーを0件にします。
3. 開始ノードから終端までプレビューします。
4. 「作品情報とアセット」で参照ファイルの欠落がないことを確認します。

エラーが残っている場合はプレイヤー用ファイルを書き出せません。

## `.yuraive`を書き出す

1. グラフタブを開き、ノードもボタンも選択していない状態にします。
2. 右側のグラフ情報で「配布」を探します。
3. 「プレイヤー用バイナリを出力」を押します。
4. ダウンロードされた`.yuraive`を作品フォルダへ置きます。

書き出したファイルには、グラフとコンテンツフォルダ内の`.star`、`.yuraive-layout.html`が含まれます。

元の`.yuraive.json`、`.star`、`.yuraive-layout.html`を配布先へ添える必要はありません。

## 素材を一緒に置く

音声、動画、画像、字幕は`.yuraive`に含まれません。

次のように、編集時と同じ相対パスで一緒に配布します。

```text
rain-story/
├── graph.yuraive
├── audio/
│   ├── opening.ogg
│   └── ending.ogg
├── images/
│   └── cover.webp
└── subtitles/
    └── opening.vtt
```

フォルダ全体をZIPなどへまとめる場合も、展開後の相対パスが変わらないようにします。

## プレイヤーで確認する

1. 配布用フォルダをAndroid版またはWindows版のライブラリへ追加します。
2. 作品の「作品情報とアセット」を開きます。
3. 参照アセットがすべて認識されていることを確認します。
4. 作品を最初から最後まで再生します。

同名の`.yuraive.json`も置いている場合、プレイヤーには`.yuraive`だけが表示されます。

JSONを修正した後は、配布用の`.yuraive`も必ず書き出し直します。
