---
title: SMBとWebDAVの問題
description: リモートフォルダまたはWindows共有へ接続できない場合に確認します。
---

リモートライブラリの問題は、入力内容、サーバー機能、端末からの到達性を順に確認します。

## SMBへ接続できない

次の項目を確認します。

- サーバー名またはIPアドレスが端末から到達できる。
- ポートが正しく、通常は`445`になっている。
- 共有名にはサーバー上のSMB共有名を入力している。
- ファイアウォールがSMB通信を許可している。
- アカウントが指定した共有とフォルダを読み取れる。
- ドメインが必要な環境では正しいドメインを入力している。

guest接続を使う場合は、ユーザー名とパスワードを両方とも空にします。

片方だけを空にした状態はguest接続になりません。

ホスト名で見つからない場合はIPアドレスでも試し、名前解決の問題か切り分けます。

## WebDAVへ接続できない

URLは`https://`で始め、認証情報、クエリ、フラグメントをURLへ埋め込みません。

ユーザー名とパスワードは専用の入力欄へ入れます。

サーバーには、次の機能が必要です。

- `PROPFIND`と`Depth: 1`によるフォルダ一覧
- Range付き`GET`による部分読み込み
- 端末が信頼できるHTTPS証明書

ブラウザで開けるURLでも、WebDAVの`PROPFIND`を許可していなければフォルダ一覧は取得できません。

一覧は見えるのにシークだけ失敗する場合は、Rangeリクエストへの対応をサーバー管理者へ確認します。

## Tailscale内のサーバーを使う

端末が同じtailnetへ接続され、名前またはIPアドレスへ到達できれば、SMBとHTTPS WebDAVを利用できます。

WebDAVはTailscale内でもHTTPSが必要です。

## WindowsのライブラリをAndroidから開けない

1. Windows版の「共有を有効化する」が有効であることを確認します。
2. Windows版で「Androidと接続」を開き、新しいQRコードを表示します。
3. Android版でQRコードを読み直します。
4. Android版のWindowsライブラリにある再読み込みボタンを押します。

この接続はWindowsとAndroidの間に直接QUIC接続を作るため、双方がUDPで直接通信できる必要があります。

企業やゲスト用のネットワークでは端末間通信が遮断されることがあるため、同じ家庭内ネットワークなどでも試します。

接続の待ち合わせサービスはメディアを中継しないため、直接経路を作れない環境では再生できません。

## 保存した認証情報を変更する

一度ライブラリから対象フォルダを外し、正しい接続情報で追加し直します。

フォルダをライブラリから外しても、サーバー上のファイルは削除されません。

認証情報はAndroid版ではAndroid Keystore、Windows版ではWindowsのデータ保護機能で暗号化して保存されます。
