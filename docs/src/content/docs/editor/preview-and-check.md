---
title: プレビューと問題を確認する
description: エディタ内で作品を再生し、エラー、警告、スクリプト実行を確認します。
---

保存や配布の前に、エディタのプレビューと問題一覧で作品を確認します。

## プレビューを開く

1. プレビューするグラフのタブを開きます。
2. 画面上部の「プレビュー」を押します。
3. 開始ノードから再生し、ボタンと遷移を最後まで試します。

プレビューでは、再生、一時停止、シーク、ボタン操作、最初からの再生を確認できます。

ボタン表示スクリプトを変更した後は、「ボタンを再評価」を押して表示を更新できます。

## デバッグ情報を見る

プレビュー上部のデバッグボタンを押すと、次の情報を確認できます。

- Script trace
- Starlarkの`print()`出力
- 実行時のコンテキスト
- プレビュー中に確定した再生履歴

プレビューの履歴はブラウザのメモリだけに保持され、エディタを閉じると失われます。

必要な場合はトレースと履歴をデバッグペインからJSON Lines形式で書き出します。

## エラーと警告を確認する

画面下のステータスバーには、エラー数と警告数が表示されます。

ステータスバーを押して「問題」を開き、各項目を選ぶと対象のノード、ボタン、スクリプト、レイアウトへ移動できます。

**エラーが1件でも残っている作品は、プレイヤーで再生できません。**

警告は編集途中の状態を含むため再生できる場合がありますが、意図した状態か確認してから配布します。

よくあるエラーは次のとおりです。

- 開始ノードがない、または複数ある。
- 遷移先のノードIDが存在しない。
- 参照する音声、画像、動画、字幕が見つからない。
- ボタンがあるのに再生設定またはレイアウトがない。
- ボタンの配置先slotがレイアウトにない。
- Starlarkの関数や戻り値が条件を満たさない。

## スクリプトだけを試す

`.star`タブを開き、実行する関数を選んでテストボタンを押すか、`Ctrl+Enter`を押します。

結果、`print()`出力、エラーは右側の「実行結果」に表示されます。

macOSでは`Command+Enter`を使います。

## 検査後に保存する

問題を修正したらプレビューを最初から再生し、開始から終端まで到達できることを確認します。

その後、変更したグラフ、スクリプト、レイアウトをそれぞれ保存します。
